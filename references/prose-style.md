# 写作规范 — Prose style & Core principles（领域无关）

这份文件是 EBR 综述正文的写作规范，**适用于任何领域**（健康 / 物理 / 食品 / 兽医 / 教育 / 经济 / 社会科学……）。
写完每一节后对照自查。除非特别标 **MANDATORY**，各条都是 rule of thumb：
该用 definitional 开头的节（术语澄清、分类定义）不必硬塞裁决句。

**用中文写正文。** 所有 prose、标题、图注、摘要都用中文；代码、DOI、citation key、公式、工具命令保持原形。

---

## 一、Core principles（六条骨架）

1. **文章结构围绕实用观点展开，证据是支撑不是骨架。**
   正文的章节标题应当是**决策性命题**（"频率加趋势比单次测量更可靠"、"在这个区间增量的边际收益已经趋零"），
   不是按证据类型或研究机制排列（不要写"§3 队列研究"、"§4 机制研究"）。
   读者关心的是"该怎么做 / 怎么想"，不是"这个领域有哪些研究"。

2. **少而精的引用。** 每一条核心论点只挑 **1–3 条**最强证据（按本领域的证据分级取 top；分级见
   `references/evidence-tiers.md`），其余 verified 条目留在 references store 里、不进正文。
   **verified ≠ 必须引用。**

3. **Abstract 必须给可操作的决策结论** —— 告诉读者应该怎么做或怎么想，
   不是罗列"研究了什么"。

4. **呈现冲突与边界。** 客观列出主要争议、竞争解释、限定人群 / 限定条件。

5. **证据不足时显式说出来**，不要用含糊措辞掩饰不确定性。

6. **教科书共识可直陈。** 当某事实是成熟教材知识、但文献库里找不到单一权威出处时，
   直接写并注明"标准教材知识"（无需 citation）。

## 二、Prose style（段落与句法层执行细则）

Core principles 规定了"标题是命题"与"摘要给结论"；下面是段落级的落地细则。

### 1. 每节首句是裁决句
读者只读第一句也能抓到本节判断。
- 反例："X 在不同条件下存在明显差异……"（描述，不下判断）
- 正例："只看这一个指标，方案 A 基本已经赢了。"（先给结论）

### 2. 数字必须带"这意味着什么"
裸数据属于资料卡，不是综述。给了数字就要说明它落在"可忽略 / 值得注意 / 足以改变决策"的哪一档。
- 反例："A 组 161，B 组 85（单位略）。"
- 正例："A 组 161 vs B 组 85——这不是'略高'，而是已经大到足以改变默认选择的优先级。"

### 3. 机制、实证、局限、实操拧成同一条推理链
不是四个并列模块。单节内部按
**判定句 → 为什么重要 → 最硬证据 → 边界 → 对选择的含义**
推进；可省略其中任一环（不是固定模板），但不要把这四件事写成四段彼此独立的材料。

### 4. 引用密度下调到段级
同一命题的 `[@key]` 只在**命题落地处**挂一次（可同时挂多个 key），
不要每个从句复述都补一遍。机器校验底线不变（每个 `[@key]` 必须对应一条 verified entry），
但视觉上应该"一段一个 anchor"。

### 5. 节奏要有长短变化
每段至少一句 **≤ 20 字的短结论句**（"更贵 ≠ 更好"、"把效果归因给单一因素不如归因给整体习惯稳妥"）。
连续三句以上都是"主句 + 括号 + 破折号 + 分号"的长复合句 → 重写。

### 6. 允许轻度评论感，禁止夸张形容词
可以写"这条推论不成立"、"这一点容易被营销夸大"；
不要写"非常厉害"、"绝对"、"炸裂"、"革命性"。**锋利用在判断上，不要用在形容词上。**

### 7. 局限节不是礼貌收尾
要写清"这条局限会**阻止什么结论**"。
- 反例："本综述的局限在于样本有限……"（套话收尾）
- 正例："只要缺少头对头的直接对照实验，'X 整体更优'就都更接近推测，而不是定论。"

### 8. 减少连续并列清单
列表只保留给：**实操建议表、错误做法清单、限定人群 / 条件、机制的三/四条通路**。
正文论证优先用 prose 推进，不要全篇都是 `- `。

### 9. 摘要不要写成编号讲义
"一、二、三、四"式 outline 在摘要里和学术报告雷同，失掉"作者声音"。
建议分 **3 段 prose**：
- 第 1 段：总裁决（"没有绝对最好，按用途/场景分"）；
- 第 2 段：按目标分流（追求 X 该选哪个、追求 Y 该选哪个）；
- 第 3 段：实践结论 + 边界。

### 10.（理工科）标题承诺的公式 / 守恒量必须真正在 prose 里推导出来
如果某节标题里写了具体的物理量 / 公式 / 守恒律 / 定理名（如 "§2 …的不变量"、"…平衡条件"、"…的反转时间公式"），
prose 里就要给该量的 **定义 + 推导（或近似论证）+ 物理意义**——不能只在括号里 fleeting 提一句名字。
理工科综述的核心价值之一就是把别人 paper 里关键定理 / 守恒量的推导讲透；标题里允诺了一个数学对象就要把它建出来。

**自查方法**：写完第一遍后，用 `grep -nE "^## "` 列出所有章节标题，
逐个核对标题里提到的"具体物理量 / 公式 / 守恒律 / 定理"——prose 是否以 `$...$` 或 `$$...$$` 形式给出了？
没有就是 underdone。这是反复出现的 fail mode，reviewer 阶段未必能抓到，作者必须在交付前自查。

### 11.（**MANDATORY**）术语 / 缩写 / 学名首次出现必须括号给中文译名
化学名、英文缩写、拉丁学名、量表 / 测定法 / 模型 / 算法名第一次出现时，**紧跟括号给中文**；
后续可直接用原名或缩写。读者不应为查一个缩写而跳出正文。

跨领域示例（首用形态）：
- 健康：URI（上呼吸道感染）、hsCRP（高敏 C 反应蛋白）、iAUC（增量曲线下面积）
- 食品：actinidin（猕猴桃蛋白酶）、psyllium（车前子壳）、DPPH / FRAP / ORAC（体外抗氧化测定法）
- 物理 / 统计：OLS（普通最小二乘）、WLS（加权最小二乘）、$\chi^2/\mathrm{dof}$（约化卡方）
- 教育 / 心理：FCI（力概念量表）、SEM（结构方程模型）、ICC（组内相关系数）

本节其它条都是 rule of thumb，**这一条是硬约束**——`lint_review.py` 不自动校验，
但交付前必须自查、reviewer 也应抽查。

## 三、数学公式排版

**数学公式必须用 LaTeX 数学环境**：行内 `$...$`、行间 `$$...$$`。
不要用 ASCII / Unicode 拼写（`σ_b² = σ²/Σ(x_i−x̄)²` 这种在 GitHub / Obsidian / Pandoc 下都很难读）。

- 下标 `_`、上标 `^`、求和 `\sum`、希腊字母 `\sigma`、矩阵 `(X'X)^{-1}` 都写正规 LaTeX。
- 变量名（$n$、$\sigma_b$、$\chi^2/\mathrm{dof}$）在中文 prose 里也用 `$...$` 包起来，统一字形。
- 仅 ASCII 决策树 / 代码块内的伪数学可保留 ASCII（代码块不渲染 LaTeX）。

> 交付提示：常见 markdown 查看器不渲染行内 `$…$`。定稿后可另出一份 HTML 交付，确保公式正常显示。

## 四、引用格式

- APA 风格。正文用 `[@citation_key]` 标记；References 节由渲染工具自动生成，不要手写。
- 图表 inline 引用时，图注注明来源 DOI 与 license；自制图导出 SVG。
- **每条观点正文引用 ≤ 3 条**（本领域一级证据 + 代表性次级证据）。同方向的复现研究只挑最强 1 条，
  其余在 §限定与争议 合并提及或仅留 references store。
- 正文 `[@key]` 必须来自本轮检索 / 分析得到的 **cite_recommend** 清单；
  **不允许凭领域常识、印象或记忆中读过的论文加引用**。
  某条标杆论文若没被本轮检索到，正确反应是"下一轮补检索"，不是手工塞一个 `[@key]` 进去。
- 撤稿（retracted）条目**禁止进正文**（渲染与 lint 都会拦）；保留在 store 里仅作透明性记录。
